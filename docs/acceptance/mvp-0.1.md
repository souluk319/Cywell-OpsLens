# MVP 0.1 Acceptance Matrix

Scope: console-shaped read-only prototype for Cywell OpsLens, with fixture-backed assistant contracts, a live OpenShift API validation lane, a Stage 1 OpenShift Lightspeed custom MCP integration contract, and the first Stage 2 alert-triggered incident analysis contract.

## Locked Goal

Prove that a dashboard-first, context-aware OpenShift operations assistant can stay out of the way of evidence while producing an evidence-first read-only analysis contract.

## Evidence Chain Extension

- AC-DASH-001 includes dashboard panels for certification readiness, catalog toolchain readiness, external runtime reviewer packet readiness, security scan/SBOM readiness, release evidence refresh, release evidence bundle readiness, and release action queue readiness. The API/UI must expose `certificationReadinessOnly`, `toolchainPlanOnly`, `reviewPacketOnly`, `scanPlanOnly`, `localEvidenceRefresh`, `bundleOnly`, `actionQueueOnly`, CLI/tooling gaps, same-HEAD freshness, command status, decision state, owner action summary, mutation flags, risk, rollback path, and missing evidence without submitting to Partner Connect, publishing catalog images, promoting drafts, mirroring/signing runtime images, or applying cluster resources. The consolidated checkpoint and roadmap Stage 5 must also carry `certificationReadiness`, `catalogToolchain`, `externalRuntimeReviewPacket`, and `securityScan` so release review sees local tooling gaps instead of relying on dashboard-only evidence.
- AC-CERT-001 treats `npm run verify:certification`, `npm run verify:catalog-toolchain`, `npm run verify:security-scan-plan`, and `npm run verify:release-refresh` as review inputs for release-manager evidence packets; they do not replace final Red Hat certification, external runtime SBOM/scan/provenance, registry digest/signature, or human approval evidence.
- `npm run verify:certification` writes `opslens.certification-readiness.v0.1`, records Community/Certified Operator packaging/doc readiness plus local `opm`/`operator-sdk` tooling gaps, verifies the certification tooling handoff document, and feeds checkpoint, roadmap Stage 5, release bundle, and release action queue evidence.
- `docs/release/evidence/external-runtime/README.md` defines external runtime reviewer roles, evidence state transitions from `example-only` to `reviewed-final`, and approval-gated mirror/sign boundaries; `npm run verify:external-runtime-plan` fails if this intake contract drifts.
- `npm run verify:security-scan-plan` writes `opslens.security-scan-plan.v0.1`, checks local scan/SBOM/signing CLI readiness, consumes same-HEAD Docker fallback runner coverage when present, lists missing vulnerability/SBOM/security-review files, separates read-only evidence generation from approval-gated `cosign sign`, and feeds checkpoint, roadmap Stage 5, and release bundle evidence. `npm run evidence:security-scan:docker` or `npm run verify:release-refresh -- --security-scan-docker` may generate owned-image vulnerability/SBOM files through digest-resolved scanner containers when local `trivy`/`syft` CLIs are unavailable, but they still create only draft review packets and no final human approval.
- `npm run evidence:external-runtime:review-packet` writes `opslens.external-runtime-review-packet.v0.1`, consolidates vLLM/Qdrant reviewer requests and approval-gated runtime commands, and feeds checkpoint, roadmap Stage 5, and release bundle evidence without promoting drafts or running mirror/sign/push commands.
- `npm run verify:release-evidence-bundle` writes `test-results/cywell-opslens-release-evidence-bundle.json` plus `test-results/cywell-opslens-release-evidence-bundle.md`; API/UI must surface the Markdown packet path as `bundle.markdownPath` while preserving `bundleOnly`, no-mutation flags, missing evidence, risk, rollback, and approval-gated command boundaries.
- `npm run evidence:release-action-queue` writes `opslens.release-action-queue.v0.1` JSON plus Markdown, groups checkpoint/release/network/external-runtime gaps by operational owner, lists read-only refresh commands and approval-gated commands not run, and feeds the dashboard as `actionQueueOnly` evidence with `actionQueue.markdownPath` without approving push, mirror, sign, install, patch, apply, delete, or scale.
- OCP handoff/action queue routing is classification-aware: `tcp-timeout`, `tcp-unreachable`, and DNS gaps route to Network/SRE; `tls-handshake-failed` routes to Cluster SRE/Security; `auth-or-rbac`, `auth-failed`, and `token-missing` route to Cluster Admin/SRE for credential or read-only RBAC evidence, backed by `npm run evidence:ocp-auth-rbac-plan`.

## Explicit Non-Goals

- No automatic `apply`, `delete`, or `scale`.
- No Kubernetes Secret fetch.
- No external LLM provider call from the prototype.
- No full Operator controller lifecycle yet; MVP 0.1 includes package contract, controller-runtime source with explicit OLSConfig patch path, TypeScript reconcile contract, and static verifier.
- No production RAG/vector database yet; Stage 1 exposes the contract, local Markdown private-RAG policy path, validate-only intake checks, and evidence export artifact.
